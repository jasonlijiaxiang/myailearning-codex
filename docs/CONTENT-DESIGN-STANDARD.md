# 售前知识库内容与构图规范

本规范适用于当前 RAG 模块及后续新增的 Agent、多模态、协议、工程保障、模型、数据与算力模块。它约束最终阅读页面，不作为页面中的说明内容展示。

## 1. 知识库语气

- 标题采用客观陈述或名词短语，例如“RAG 适用性判断”“RAG 双链参考架构”。
- 避免把编辑者与读者的对话写进标题：不用“先确定”“再判断”“我们来看”“你需要”“不要一上来”等表达。
- 流程步骤可以使用动词，但必须描述系统动作或客户动作，而不是编辑指令。
- 正文以中文为主；专业术语首次出现提供准确英文或通行缩写。

## 2. 信息层级

- 一级模块标题说明主题；二级标题说明知识对象；卡片标题说明单一概念。
- “重要边界、风险、错误前提”使用高对比警示组件；普通补充说明不得与其争夺视觉层级。
- 编辑规则、语言策略、构建说明等内部信息不进入读者页面。
- 同一概念只设一个主要归属模块；其他模块给出必要的局部解释和回链。

## 3. 动态均衡卡片

- 卡片数量来自实际内容，不把 8 张或其他数量写成模板前提。
- 桌面端按当前数量自动计算完整行，每行最多 4 张，各行数量差不超过 1。例如 5 张为 3+2、6 张为 3+3、7 张为 4+3、9 张为 3+3+3。
- 平板采用双列；单数时最后一张占满最后一行，避免孤立空位。手机采用单列。
- 不使用 `auto-fit` 任由浏览器产生 5+3、4+1 等残缺构图；布局算法必须覆盖单数、双数、空列表与未来增减。

## 4. 标题与换行

- 数字与量词、英文缩写与中文解释、专有名词不得被任意拆开。
- 复杂标题拆成“主标题 + 统计单元 / 副标题”，不依赖浏览器在任意字符处换行。
- 分隔符必须与后一语义单元绑定，换行时不得单独留在上一行或下一行。
- 使用 `white-space: nowrap` 保护短语，只在设计好的单元边界换行。
- 标题在常见桌面宽度下不应出现只有 2–4 个汉字的孤行。

## 5. 模块复用规则

每个新模块沿用以下骨架，并按内容需要增减：

1. 模块定位与相关模块
2. 基础概念与核心原理
3. 机制、架构与变体
4. 适用边界与选型
5. 云服务连接与方案组合
6. PoC、评估、安全与运营
7. 客户问题实战包
8. 来源、证据等级与复核日期

结构是质量检查清单，不是数量配额。没有价值的图、卡片或说明不应为了对齐模板而添加。

## 6. 发布前检查

- 内容：标题是否为知识库语气；术语是否中英对应；结论是否有边界。
- 构图：固定卡片是否对称；标题是否按完整语义单元换行；视觉重点是否明确。
- 响应式：桌面、平板、手机的列数和阅读顺序是否清晰。
- 导航：知识地图入口、跨模块回链和页内锚点是否有效。
- 维护：来源、核验日期和模块归属是否可追踪。
