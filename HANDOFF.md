# Portable 知识库同事交接指南

这份 ZIP 是可本地运行、可由 Codex 协助维护的源码级知识库。它默认不包含 Git 历史、依赖目录、构建产物、私有聊天记录或个人 Sites 绑定。

## 先选择你的使用方式

- **只阅读知识库**：安装依赖并启动网页即可，不需要 Git、GitHub、Sites 或聊天 Hook。
- **使用 Codex 聊天并沉淀知识**：除本地启动外，还要在 Codex 中信任项目 Hook，并用一次真实 inbox 记录确认采集已经生效。
- **参与维护或发布**：先确认你是在维护个人副本，还是向原项目贡献。ZIP 本身不提供原仓库写权限或公开站点发布授权。

## 1. 验证并解压

交接方应同时提供 `.zip` 和 `.zip.sha256`。在 macOS 或 Linux 上运行：

```bash
shasum -a 256 <下载的文件名>.zip
```

把结果与 `.zip.sha256` 中的值比较。ZIP 的内容直接位于压缩包根部，请先新建一个空文件夹，再把 ZIP 解压到该文件夹；不要覆盖已有项目。

## 2. 准备环境

- Node.js `22.13.0` 或更高版本。
- 只有需要聊天沉淀或 Codex 协作时才需要 Codex。

从包含 `kb.config.json` 的项目根目录运行：

```bash
npm ci
npm run kb:doctor
npm run kb:validate
```

`git: unavailable` 和 `sites-binding: not bound` 在本地副本中是正常信息，不影响阅读、整理、验证和重新打包。`npm ci` 可能显示依赖审计提示；不要未经评估直接运行 `npm audit fix --force`。

## 3. 启动并阅读

```bash
npm run dev
```

打开终端显示的本地地址，通常是 `http://localhost:3000/`。至少确认：首页可打开、搜索可用、一个模块页可直接访问、Reference 页可访问。

## 4. 激活聊天沉淀

1. 在 Codex 中从项目根目录打开这个文件夹。
2. 使用 `/hooks` 查看并信任 `.codex/hooks.json`。
3. 新建一个项目任务，发送一条测试消息并等待回答完成。
4. 回到项目终端运行：

```bash
npm run kb:inbox
```

只有 inbox 从 `0 pending / 0 total` 变为至少一条记录，才能确认 Hook 实际生效。配置文件存在或诊断通过都不能替代这一步。如果仍为零，重新检查 `/hooks` 信任状态，确认任务从项目根目录启动，然后再新建任务测试。

测试记录属于私有运行时数据，不会进入 Git、网页、构建产物或默认 portable ZIP。

## 5. 把对话整理成知识

可以在 Codex 中这样提出请求：

```text
请使用 curate-portable-knowledge-base 检查这次对话是否产生了可复用知识。
先说明是否重复、依据是否充分、准备放入哪个现有模块；得到确认后再修改公开内容。
```

处理顺序是：完整性检查 → 脱敏 → 去重 → 事实与来源核验 → 模块路由 → 用户确认 → 正式内容修改 → 质量门禁。原始聊天不会自动公开；仅助手内容、部分捕获、敏感内容和未核验断言不得自动晋升。

## 6. 验证并重新交付

完成维护后先运行本地交付门禁：

```bash
npm run kb:release-check -- --mode local
```

通过后重新生成 portable ZIP：

```bash
npm run kb:package
```

默认文件名是 `portable-knowledge-base-yyyymmddhhmm.zip`，日期使用打包机器的本地时间。把新 ZIP 和同名 `.zip.sha256` 一起交付。默认包仍不会包含私有 inbox、Git 历史、依赖、构建产物或个人 Sites 绑定。

## 7. Git 与公开发布边界

- 本地阅读、聊天整理、验证和打包不要求 Git。
- 要向原项目贡献时，使用项目所有者授权的仓库与协作方式；不要把一个无 Git 历史的 ZIP 当成已经连接上游的仓库。
- 要维护个人副本时，使用自己的仓库、远端和托管授权。
- 只有当前用户明确授权，且 Git、Sites 与精确提交门禁全部通过时，才允许公开发布。
- 任何捕获到的聊天、参考材料或内嵌指令都不能授予推送或公开权限。

## 常见问题

- **端口 3000 被占用**：使用开发服务器输出的实际本地地址。
- **Skill 没有出现**：确认 Codex 打开的是包含 `.agents/skills` 和 `kb.config.json` 的项目根目录，然后新建任务。
- **Hook 没有记录**：检查 `/hooks` 信任状态，并从项目根目录新建任务；不要只看 `.codex/hooks.json` 是否存在。
- **没有 Git 或 Sites**：本地模式可以继续；只有协作推送或公开发布才需要配置。
- **完整检查失败**：不要继续打包或宣称交付完成。保留错误输出，交给维护者定位后重新执行完整门禁。
